Event-Handler mit on

Einen Event-Handler kann man erzeugen, indem man dem Namen des Events den Begriff on voranstellt. Jedem Element kann nur ein Event-Handler zu einem jeweiligen Event zugeordnet werden.

<input value="Drück mich!" onclick="function()" type="button"> in html möglich
absatz.onmouseover = nameFunction; in script möglich, Funktionsaufruf ohne Klammern

Event-Listener

Es können mehrere Event-Listener zu einem Event einem Element zugeordnet werden. In Klammern kommt erst das Event in Anführungszeichen und dann der Name der Funktion, die ausgeführt werden soll, ohne Klammern. Alternativ kann eine anonyme Funktion eingeführt werden mit dem Funktionsblock in geschweiften Klammern. Diese kann später aber nicht entfernt werden. Mit removeEventListener wird der Event-Listener wieder entfernt.

canvas.addEventListener("click", nameFunction); Mausklick-Event
canvas.addEventListener("mousemove", function(event) { }); Mausbewegungs-Event
canvas.addEventListener("contextmenu", () => { }); rechter Mausklick
canvas.removeEventListener("click", nameFunction); entfernt Event-Listener

Zugriff auf Element, das Event auslöst

event.target

Beispiele:

event.target.style.backgroundColor = "green"; ändert Hintergrundfarbe
event.target.closest("p"); Zugriff auf das p-Element, das das auslösendeElement umfasst

Bubbling

Wenn ein Element durch umschließende Elemente im Bereich mehrerer Events liegt, werden durch die Aktion alle Events ausglöst. Wenn man das nicht möchte, kann der Befehl event.stopPropagation() genutzt werden. Dann wird nur das "innerste" Event getriggert.

Tastaturereignisse

Shift- und Pfeiltasten lösen keydown und keyup-Events aus, aber kein keypress-Event. Um zu sehen welche Taste gedrückt wurde, wird das keyword-Event mit den Attributen key und code benötigt.

function tastatureingabe(e) {
    alert("Key: " e.key + " Code: " + e.code);
}
element.onkeypress = tastureingabe;

Ausgabe: Key: w Code: KeyW

key beachtet Groß- und Kleinschreibung, code nicht. Es gilt die amerikanische Tastaturbelegung.
