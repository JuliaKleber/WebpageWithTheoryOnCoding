Git ist eine freie Software zur verteilten Versionsverwaltung von Dateien, die durch Linus Torvalds 2005 initiiert wurde.

GitHub hat über 80 Mio Nutzer und ist von Microsoft.

Begriffe

Working Tree / untracked area Dort arbeitet man gerade.
Branches
Staging Area hier sind die Commits
commits gespeicherte Änderungen
Pull Request Informiert andere Entwickler darüber,
dass es einen Branch mit Änderungen gibt.

Befehle

git init [repository name] Erstellt ein leeres Git Repository.
git remote add origin [repository url] Erstellt Verknüpfung zum remote Repository
git push -u origin master Lädt Dateien ins remote repository.
git status / gst Zeigt an, was sich im Working Tree befindet.
git add Überträgt die Änderungen aus dem lokalen Repository zur Staging Area.
git commit -m "Kommentar" Überträgt den Snapshot aus der Staging Area ins remote Repository.
git pull Damit kann man einen Branch aus einem remote Repository lokal speichern.
git merge Führt verschiedene Branches zusammen.
git config -global user.name "[name]" Definiert den Namen des Urhebers der Commits.
git config -global user.email "[email address]" Definiert die Email-Adresse, die mit den Commits genutzt werden soll.
git clone [SSH] Kopie eines Repositories wird erstellt.
git reset [file] Änderungen werden rückgängig gemacht.
git reset [commit] Änderungen werden rückgängig gemacht.
git checkout [branch name] Wechselt zwischen verschiedenen Branches.
git checkout -b [branch name] Erstellt neuen Branch und wechselt zu diesem.
git log zeigt die bisherigen commits an
git diff commitname zeigt die Änderungen
git branch -a zeigt alle Branches an
gh browse öffnet github repo im Browser

1. Commit zu GitHub

git init Erstellt ein leeres Git Repository.
git add * / git add . Fügt alle Dateien im Verzeichnis hinzu.
git commit -m "first commit" speichert Änderungen lokal
gh repo create --public --source=. erstellt github repo
git branch -M main nennt master in main um
git remote add origin [url] wenn das repo auf der github page erstellt wurde
git push -u origin main Bei Passwort das Token eingeben.

2. Commit zu GitHub

git add . Fügt alle Dateien im Verzeichnis hinzu
git commit -m "second commit" Änderungen werden lokal gespeichert
git push origin master Die Änderungen werden auf github (origin) gespeichert
